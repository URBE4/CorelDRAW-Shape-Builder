Восстановление макроса после вылета Corel или отката GMS
=====================================================

Почему «возвращается» к старому виду после вылета:
  GlobalMacros в памяти теряется при падении Corel. На диск попадает только то,
  что вы сохранили в VBA (Файл — Сохранить GlobalMacros). Если не сохраняли —
  при следующем запуске подгружается последняя сохранённая копия (часто старая,
  без вкладок и с коротким заголовком «Shape Builder»).

После deploy_direct.ps1:
  В CorelDRAW у VBProject через COM часто нет Save() — в консоли будет «Save() failed»:
  это нормально. Всё равно сразу Alt+F11 → Файл → Сохранить GlobalMacros (иначе при вылете
  подтянется старый .gms с диска).
  Скрипт дополнительно пишет: vba_backup\deploy_<дата-время>\ (копия .bas + UserForm1_Code.txt)
  и vba_export_last\ (экспорт из VBE: Module1.bas, UserForm1.frm + при форме может быть .frx).

Порядок этапов разработки: см. WORK_ORDER.txt в этой папке.
Основная среда: CorelDRAW Graphics Suite 2026 (см. deploy_all.ps1, ShapeBuilder_CorelConnect.ps1).

Источник правды — файлы в этой папке:
  CorelDRAW_Furniture_Facades_Macro.bas  (Module1)
  UserForm1_Code.txt

В заголовке формы: Shape Builder Pro v0.1.x  [2026-layout-LR] — SB_CODE_REV из UserForm1_Code.txt
  Блок «Параметры фасада» и серая строка про deploy на «Основное» убраны.
  Если видите v2.2.0, «Пневм» и т.д. — это не код из этой папки; см. VERIFY_UI.txt
В Immediate: ?Module1.MEBEL_MACRO_VERSION(0)  -> 0.1.5 после последнего деплоя

ВАЖНО — почему «ничего не менялось»:
  В списке макросов был Module2.RunModernShapeBuilder — это СТАРЫЙ дубликат.
  Запускали его — открывалась старая форма. Актуальный код только в Module1.
  deploy_direct.ps1 теперь удаляет Module2. Запускайте: Module1.Фасады
  (в Module1 публичный вход только Фасады; остальное — Private).

Шаги:
1) Запустите CorelDRAW, откройте любой документ.
2) Сервис / Параметры — безопасность VBA — включите доступ к объектной модели VBA (доверие к проекту).
3) Один раз нажмите Alt+F11 (открыть редактор VBA), при необходимости закройте его.
4) Заливка: deploy_all.ps1 (сам запускает Corel при необходимости и deploy_direct.ps1, лог deploy_last_log.txt)
   или deploy_direct.ps1 вручную; двойной щелчок ЗАПУСК_ДЕПЛОЯ.bat вызывает deploy_all.ps1.
5) В редакторе VBA: Файл — Сохранить GlobalMacros (или закройте VBA с сохранением).

Если скрипт пишет VBE unavailable — повторите шаги 2–3 и снова deploy.

Пустой список в «Сервис — Макросы — Запустить», проект GlobalMacros:
  Значит в GlobalMacros нет кода (или не сохранён .gms). Макросы появятся только после
  заливки Module1 (публичный Sub Фасады; остальное — Private).
  1) Закройте окно «Запустить макрос». 2) Откройте папку проекта с deploy_direct.ps1.
  3) Запустите deploy_direct.ps1 в PowerShell (ПКМ — «Выполнить с помощью PowerShell»
     или: cd в папку, затем .\deploy_direct.ps1). Должно быть: Module1 OK, UserForm1 OK,
     DIRECT DEPLOY OK. 4) Alt+F11 → Файл — Сохранить GlobalMacros. 5) Снова Сервис —
     Макросы — в списке выберите Module1.Фасады.
  Если deploy пишет ошибку — пришлите текст из консоли. Скрипт сначала цепляется к
  уже открытому Corel через CorelDRAW.Application (без номера версии), затем к .28…21 (2026 — ShapeBuilder_CorelConnect.ps1).

Резервная копия: периодически экспортируйте Module1 и UserForm1 из VBA в файлы .bas / форму вручную.
